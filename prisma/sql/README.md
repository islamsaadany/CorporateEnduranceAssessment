# Hand-runnable SQL scripts

These files are auto-generated mirrors of the Prisma schema and seed script. They exist for users who don't have a local Node/terminal environment and need to apply changes to Neon directly via the Neon SQL editor.

## How to use

Open the [Neon Console](https://console.neon.tech) → your project → **SQL Editor**. Paste each file's contents into a query, run it, and check for errors.

| Order | File | What it does | When to run |
|-------|------|--------------|-------------|
| 1 | [`000_initial_schema.sql`](./000_initial_schema.sql) | Creates all enums, tables, indexes, foreign keys | First-time setup, or when re-pointing at a fresh database |
| 2 | [`001_seed_sample_data.sql`](./001_seed_sample_data.sql) | Inserts super admin + Settings singleton + 1 sample assessment with 5 respondents | After 000, or whenever you want to refresh sample data |

## Sample super-admin login

After running `001`, log in to the admin panel with:

- **email:** `superadmin@forefront.example`
- **password:** `change-me-on-first-login`

Change this password in the admin panel (or by re-generating the seed with new env vars) before any non-local use.

## How these files are generated

| File | Source | How |
|------|--------|-----|
| `000_initial_schema.sql` | `prisma/schema.prisma` | `npx prisma migrate diff --from-empty --to-schema-datamodel prisma/schema.prisma --script` |
| `001_seed_sample_data.sql` | `scripts/gen-seed-sql.mjs` (mirrors `prisma/seed.ts`) | `node scripts/gen-seed-sql.mjs > prisma/sql/001_seed_sample_data.sql` |

If you have a local Node environment, prefer the npm scripts (`npm run db:setup`) — they are equivalent but pick up live changes automatically. The hand-runnable SQL files are regenerated by Claude Code sessions whenever the schema or seed changes.

## Future schema changes

When the schema changes, a new file (`002_…sql`, `003_…sql`, …) will be added here containing **only the diff** from the previous version, plus, where relevant, a data-migration block. Each file is meant to be run exactly once in order. The `000_initial_schema.sql` file is regenerated from scratch only when starting against a fresh database.
